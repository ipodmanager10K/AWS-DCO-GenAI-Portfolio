## 2. 내가 만든 DCO 인프라 용어 정의

### Rack

1. 쉬운 설명: 수많은 IT 장비를 깔끔하게 정리해서 꽂아두는 '철제 조립식 책장'이나 '대형 서랍장'입니다.

2. DCO 직무 이해에서 어디에 나올 수 있는지: 가상의 작업 요청서(Ticket)에서 작업 대상 위치를 표기할 때 주로 등장합니다. (예: EDU-RACK-A01에 있는 장비를 점검해 주세요.)

3. 학생이 자기 말로 바꿔 쓸 때 참고할 표현: "장비들을 모아두는 아파트형 거치대", "데이터센터의 주소 같은 철제 책장"

4. 주의할 점: 단순히 가구처럼 세워두는 것이 아니라, 전력 공급선과 랜선이 복잡하게 얽혀 있는 전기·통신 시설이라는 점을 기억해야 합니다.

### Server

1. 쉬운 설명: 우리가 집에서 쓰는 컴퓨터에서 모니터와 키보드를 빼고, 본체의 성능만 엄청나게 키워놓은 '고성능 컴퓨터'입니다. 24시간 내내 켜져서 웹사이트나 앱의 데이터를 처리합니다.

2. DCO 직무 이해에서 어디에 나올 수 있는지: 부품 결함 수리(Break-Fix) 가상 시나리오에서 가장 자주 만나는 핵심 장비입니다.

3. 학생이 자기 말로 바꿔 쓸 때 참고할 표현: "화면 없는 초고성능 컴퓨터", "클라우드 서비스가 실제로 돌아가는 물리적 본체"

4. 주의할 점: 집에서 쓰는 PC와 달리 얇고 넓적한 상자 모양(랙에 서랍처럼 슬라이딩으로 끼우는 형태)으로 생겼다는 점을 염두에 두면 좋습니다.

### NIC

1. 쉬운 설명: 컴퓨터가 인터넷(네트워크) 세상과 연결될 수 있도록 랜선을 꽂는 '랜 카드' 또는 '네트워크 통신용 구멍'입니다.

2. DCO 직무 이해에서 어디에 나올 수 있는지: 특정 서버의 통신이 불안정할 때 하드웨어 결함인지 선 연결 문제인지 분류하는 가상 시나리오에 등장합니다.

3. 학생이 자기 말로 바꿔 쓸 때 참고할 표현: "서버의 인터넷 연결 현관문", "랜선 꽂는 통신 부품"

4. 주의할 점: 데이터센터에서는 일반 랜선(구리선)뿐만 아니라 빛으로 통신하는 광케이블용 NIC도 많이 사용하므로 모양이 다양할 수 있습니다.

### ToR Switch

1. 쉬운 설명: 랙(철제 책장) 맨 위층에 주로 설치되는 '대형 멀티탭 형태의 인터넷 공유기'입니다. 그 랙에 꽂힌 수십 대의 서버 랜선을 이 스위치에 모아서 외부 인터넷망과 연결해 줍니다.

2. DCO 직무 이해에서 어디에 나올 수 있는지: 랙 전체의 서버들이 동시에 통신이 안 될 때, 이 상위 스위치 장비의 설정을 확인하거나 케이블을 점검하는 단계에서 언급됩니다.

3. 학생이 자기 말로 바꿔 쓸 때 참고할 표현: "랙 전용 대형 인터넷 공유기", "서버들의 통신 신호를 모아주는 대장 장비"

4. 주의할 점: 가정용 공유기와 달리 수십 개의 포트(구멍)가 있고, 대용량 데이터를 아주 빠른 속도로 처리하는 기업용 전문 장비입니다.

### Link Down

1. 쉬운 설명: 쉽게 말해 '인터넷 연결 끊김' 상태입니다. 케이블이 빠졌거나, 장비 전원이 꺼졌거나, 부품이 고장 나서 통신 신호가 완전히 안 잡히는 현상입니다.

2. DCO 직무 이해에서 어디에 나올 수 있는지: 모니터링 시스템에서 가상의 경고 로그가 발생했을 때 티켓에 적히는 가장 흔한 문구 중 하나입니다. (예: DEMO-SERVER-03 NIC 1번 포트 Link Down 발생)

3. 학생이 자기 말로 바꿔 쓸 때 참고할 표현: "인터넷 선 뽑힘 상태", "통신 신호 두절"

4. 주의할 점: 하드웨어가 완전히 망가졌을 때도 발생하지만, 단순히 케이블이 살짝 헐거워져서 발생할 수도 있으므로 단계별 확인이 필요합니다.

### CRC error

1. 쉬운 설명: 통신 케이블을 통해 데이터를 주고받는 과정에서, 선이 노후화되었거나 주변 간섭 때문에 데이터가 깨지거나 뭉개져서 도달하는 '통신 데이터 배달 사고(오류)'입니다.

2. DCO 직무 이해에서 어디에 나올 수 있는지: 연결은 되어 있는데 속도가 너무 느리거나 간헐적으로 패킷(데이터 조각)이 유실될 때, 케이블 교체나 포트 청소를 판단하는 근거로 나옵니다.

3. 학생이 자기 말로 바꿔 쓸 때 참고할 표현: "전송 중에 찌그러진 데이터 에러", "케이블 불량으로 인한 통신 잡음"

4. 주의할 점: Link Down처럼 선이 아예 끊어진 게 아니라, 연결은 되어 있는데 데이터가 깨지는 상태라는 차이점을 명확히 구별해야 합니다.

### SSD failure

1. 쉬운 설명: 데이터를 저장하는 장치인 SSD가 수명을 다했거나 고장 나서 데이터를 읽고 쓸 수 없는 '저장장치 고장' 상태입니다.

2. DCO 직무 이해에서 어디에 나올 수 있는지: 표준 운영 절차(SOP)를 보며 가상의 고장 난 하드디스크/SSD를 새 부품으로 추출하고 교체하는 단골 Break-Fix 실습 시나리오입니다.

3. 학생이 자기 말로 바꿔 쓸 때 참고할 표현: "저장 공간(메모리) 사망", "데이터 기록 부품 고장"

4. 주의할 점: 실제 환경에서는 데이터 유실을 막기 위해 여러 개의 저장장치를 묶어두는 기술(RAID)을 쓰기 때문에, 고장 난 즉시 서비스가 터지지는 않지만 방치하면 안 되는 경고입니다.

### Fan Alert

1. 쉬운 설명: 서버 내부에 열을 식혀주는 '선풍기(냉각 팬)'가 먼지가 끼었거나 모터가 고장 나서 제대로 안 돈다는 '냉각 팬 경고'입니다.

2. DCO 직무 이해에서 어디에 나올 수 있는지: 서버가 과열되어 자동으로 꺼지기 전에, 미리 부품을 교체하도록 알려주는 사전 예방 정비 티켓에서 자주 볼 수 있습니다.

3. 학생이 자기 말로 바꿔 쓸 때 참고할 표현: "서버 내부 선풍기 고장 경보", "열 식히는 쿨러 이상 신호"

4. 주의할 점: 팬 하나 고장 났다고 컴퓨터가 바로 멈추지는 않지만, 데이터센터 내부 온도를 일정하게 유지하는 것은 장비 수명과 직결되므로 중요하게 다루어집니다.
